ARTIST PIPELINE DOCS: Character Creation (and use of Cal3d/ReplicantBody)
Rbody Tips and Example File (Using 3DS MAX)

Here is a demo rbody file created by Delta3d for you to look at. Below is some referance information that may prove helpful in creating your own. This file did not use blended animations or any other advanced features, and was meant to be a simple pipeline test. 

# ReplicantBody v0.1

path "./marine-1"
skeleton "Skeleton.CSF"
default_act "ACT_STAND"
scale 1.00
foot_offset 0.15 ## Before scaling

## Bones used when calculating character speed
ContactBones {
   bone_name "Bip02 L Foot"
   bone_name "Bip02 R Foot"
}

## Bones from which you can get the direction
DirectionBones {
   head_bone "Bip02 Head"
}

Animation {
   name "Breath"
   filename "Breath.caf"
   act_name "ACT_STAND"
   act_weight 1.0
   speed 0.0
   is_looped 1
}

Animation {
   name "Walk"
   filename "Walk.caf"
   act_name "ACT_WALK"
   act_weight 1.0
   speed 2.0
   is_looped 1
}

Animation {
   name "Run"
   filename "Run.caf"
   act_name "ACT_RUN"
   act_weight 1.0
   speed 100.0
   is_looped 1
}

Animation {
   name "LowWalk"
   filename "LowWalk.caf"
   act_name "ACT_LowWalk"
   act_weight 1.0
   speed 1.0
   is_looped 1
}

########
# 4 meshes attached to 1 skeleton
########
Mesh {
   name "HEAD"
   filename "HEAD.CMF"
   is_default 1
}

Mesh {
   name "BODY"
   filename "BODY.CMF"
   is_default 1
}

Mesh {
   name "M16m80"
   filename "M16m80.CMF"
   is_default 1
}

Mesh {
   name "Helmet"
   filename "Helmet.CMF"
   is_default 1
}

#####
# 4 materials from cal3d
#####
Material {
   filename "HEAD.CRF"
}

Material {
   filename "BODY.CRF"
}
Material {
   filename "M16m80.CRF"
}

Material {
   filename "Helmet.CRF"
}
######
# 4 ids but only 3 material ids
######
MaterialSet {
   name "body"    ### note: this is NOT the "BODY" mesh but name you call the material set
   is_default 1
   Connection {
      thread_id 0
      material_id 0
   }
   Connection {
      thread_id 1
      material_id 0
   }
   Connection {
      thread_id 2
      material_id 3
   }
   Connection {
      thread_id 3
      material_id 2
   }
}


Some tips on this rbody script.

The model was made of 4 seperate meshes: head, body, helmet, and m16m80. All meshes were attached to a single skeleton by skinning in 3dsmax7, in this case a biped was used but it was still skinned and not physiqued.

Two objects had the same material (the head and body) and the helmet and m16 each had their own.

Here are some notable items in mind.

    1) There must be the same number of material connections as meshes in the object.
    2) Even though head and body used the same texture both need .crf files. this is how rbody keeps track of what texture is on what object.
    3) The thread_id is basically a number assigned for a mesh object in the set. The order starts from 0 based on how your objects are ordered in your max 'edited named selection set'.
    4) It's sometimes easiest to keep Max open once you output your mesh so you can reference the numbers assigned to your materials and their order. These are your material_id numbers for use cal3d and in rbody. You can number your materials in Max, if you like, with the format 'texture name [#]' where the [#] is your material_id number.
      NOTE: We did still find times when id's did not corrispond correctly. if this happens simply try changing your material_id #'s in the rbody script. (reason unknown)
    5) In the above example...the material set name 'body' is not the 'BODY' mesh. Just a coincidental naming convention on this model. (in hindsight...it might have been better to name that something else to avoid confusion)
    6) The 'MaterialSet' is a set of all the materials assigned to meshes attached to the skeleton, and though no name shown it is still in same order as above. So the first one is for the HEAD the second for BODY. Which is why the material_id is the same as they used the same material in max.
    7) Even though we didn't do it for this example, you might find it helps to remember your thread_id by listing your rbody meshes in the order they appear in your max named selection set.
    8) Remember to place all you .caf animation files being used into the same folder with your mesh and skeleton files.
