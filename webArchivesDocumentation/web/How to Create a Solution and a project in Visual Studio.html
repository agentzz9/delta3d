How to Create a Solution and a project in Visual Studio

Admin Friday, November 05 2004 @ 09:47 PM UTC Views: 40,560
Now we will start setting up the VS environment. First create a Visual Studio blank solution, which all the tutorials will be put under. However, if you prefer you can set one solution for each project.

To create a blank solution, in the VS main menu bar select:

    File-> New -> Blank Solution

When you do this, the following window will appear:

After you have named your solution and chosen the location for it, click OK.

Once you've done this, your solution will appear in the Solution Explorer on one side of the VS window. In the Solution Explorer, right click on the solution name and select:

    Add -> New Project

In the new project window select "Visual C++ Projects" and "Win32 Console Project" and choose a name for your first project.

If your Visual Studio IDE has the default settings the application wizard will create a project similar to this:

Now let's use the environmental variables created and setup the project for Delta3D:

How to setup the include files for a Delta3D project in VS

First we will use the environmental variable $(DELTA_INC) created during installation to setup the project for Delta:

Right Click on your project in the VS Solution Explorer and select "Properties", or in the global VS menu bar, select:

    Tools->Options->Projects


A window similiar to this will appear:



Select "C/C++" and make the following changes:

Under "General" set the "Additional Include Directories" to: $(DELTA_INC)



Under "Code Generation" and change "Runtime Library" to: Multi-threaded DLL(/MD)



Under "Language" set "Enable Run-Time Type Info" to : Yes(/GR)



Under "Precompiled Headers" set "Create/Use Precompiled Header" to: Not Using Precompiled Headers



The other options remain at the default values. You can look at "Command Line" under "C/C++", and the command line will look similiar to the following:

[code] /Od /I "C:\DELTA\delta3d\inc" /I "C:\DELTA\delta3d\ext\inc" /D "WIN32" /D "_DEBUG" /D "_CONSOLE" /D "_MBCS" /Gm /EHsc /RTC1 /MD /GR /Fo"Debug/" /Fd"Debug/vc70.pdb" /W3 /nologo /c /Wp64 /ZI /TP [/code]

Now let's go for the next step and set the library files:

How to setup the library files for a Delta3D project in VS

Now we will use the environmental variable $(DELTA_LIB) created in the installation to setup the project for Delta3D:

Right click on your project and select "Properties" or, in the global VS menu bar, select:

    Tools->Options->Projects

A window like this will appear:


Select "Linker" and in "General" set "Additional Library Directories" to: $(DELTA_LIB)



The others options won't change from the default values. You can select "Command Line" and your command line will became similar to this :
[code] OUT:"Debug/Project1.exe" /INCREMENTAL /NOLOGO /LIBPATH:"C:\DELTA\delta3d\lib" /LIBPATH:"C:\DELTA\delta3d\ext\lib" /DEBUG /PDB:"Debug/Project1.pdb" /SUBSYSTEM:CONSOLE /MACHINE:X86 kernel32.lib user32.lib gdi32.lib winspool.lib comdlg32.lib advapi32.lib shell32.lib ole32.lib oleaut32.lib uuid.lib odbc32.lib odbccp32.lib [/code]

Now we are ready to clean-up the default VS project.

We will not need some of these files and, to make things simpler, we will get rid of them. First, in the Solution Explorer window, click on the "+" by the Source Files and the Header Files to display their contents if not already displayed. Right click on sdtafx.cpp select "Remove". Repeat this for stdafx.h and ReadMe.txt.

In the main editing window open the Project.cpp file remaining, delete the "#include stdafx.h" line and edit the main function so it reads:

[code] int main() { return 0; } [/code]


Now save it.

Your project will look similar to this:


That's the end of the Installation and Configuration section. The remaining tutorials can be done in any order. I would suggest you head over to the HelloWorld tutorial and start to code!
